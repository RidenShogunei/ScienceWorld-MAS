# MrlX-like MGRPO Adaptation Notes

This repo's original V7 joint MGRPO script was a conservative ScienceWorld
adaptation: Main rollout was greedy, Sub was sampled, group size was 4, reward
included local progress/action-validity shaping, and the clip range was narrow.

The MrlX source uses a different training shape:

- GRPO samples multiple completions per prompt, commonly 8.
- Rollout sampling is high-temperature.
- KL/reference penalty is effectively disabled in the released scripts.
- Sub-agent data is generated by Main-agent interaction and trained with a
  reward inherited from the Main outcome, gated by format.

`scripts/run_joint_mgrpo_expert_subtask_contract_v3_mrlx_like.sh` is an
ablation that moves our ScienceWorld trainer toward that shape:

- Main and Sub both sample during rollout.
- `MGRPO_GROUP_SIZE` defaults to `8`.
- `--rollout-temperature 1.0` and `--rollout-top-p 0.95`.
- `--clip-low 0.2 --clip-high 0.28`.
- `--beta 0.0`.
- `--sub-reward-mode rollout_format`.
- Local progress/action-validity/repetition shaping is disabled by default.

This is still not a full MrlX reproduction. MrlX relies on slime/verl plus an
asynchronous DB path for Sub-agent samples, while this repo keeps the local
ScienceWorld rollout/update loop. The goal of this variant is to test whether
the earlier weak RL result came from overly conservative sampling and local
reward shaping before investing in a larger infrastructure rewrite.
